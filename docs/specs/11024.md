# Spec: Markdown Hyperlinks in Text Elements (#11024)

## Goal

A lone markdown link `[label](url)` in a text element SHALL render on the canvas as hyperlink-styled `label` that opens the normalized URL through the existing link pipeline (`handleElementLinkClick`, `onLinkOpen`, `normalizeLink`).

## Context

- **Issue:** [excalidraw/excalidraw#11024](https://github.com/excalidraw/excalidraw/issues/11024)
- **Current behavior:** `[label](url)` is shown as literal characters until parsed; primary interaction for links was the small link icon.
- **Expected:** Stored scene uses visible `label` plus `element.link`; canvas and SVG show link styling; when the text element is not selected, pointer hits on the text bounding box activate the link (same semantics as existing hyperlink handling for opening targets).

## Component / Module

| Area | Path |

## Changes Required

1. **`markdownLink.ts` (common):** Parse at most one `[label](url)` per full string; reject unsafe schemes (`javascript:`, dangerous `data:`, etc.); expose `parseMarkdownLink` for App and element layer.
2. **`App.tsx`:** On text commit, if `parseMarkdownLink` matches, set `text` to `label`, `link` to normalized URL, optional toast.
3. **`textHyperlink.ts`:** Resolve URL from `element.link` or lone markdown; derive lines + `styleAsHyperlink` for measure/render.
4. **`renderElement.ts`:** When `styleAsHyperlink`, use hyperlink color + per-line underline; preserve non-link appearance otherwise.
5. **`helpers.ts`:** For text with resolved URL and element not selected, treat bbox hit as link hit (icons unchanged for non-text).
6. **`staticSvgScene.ts`:** Match canvas hyperlink appearance; wrap with `<a>` where `element.link` does not already provide an outer anchor.
7. **Tests:** Unit tests for parser; manual QA for click/drag threshold and selection-vs-link (marquee / click outside text).

## Non-goals

- Full Markdown (bold, lists, etc.) in text elements.
- Changing embeddable elements or non-text `link` behavior beyond compatibility with shared helpers.
- **Selection UX trade-off:** Opening the link from the label when unselected implies selecting the element by a single click on the label is harder; users may use marquee or click outside the glyph box. WYSIWYG text editing while focused is unchanged.

## Acceptance Criteria

1. After submit, a single valid `[label](url)` becomes stored `text === label` with truthy `element.link` (sanitized).
2. Canvas shows underline + distinct link color for resolved hyperlinks; plain text unchanged.
3. With element not selected, pointer down/up on the text bbox within drag threshold opens the URL via existing link handling (tab / `onLinkOpen` behavior unchanged from product defaults).
4. SVG export reflects hyperlink styling consistent with canvas.
5. Malicious schemes in markdown are not promoted to navigable links.
