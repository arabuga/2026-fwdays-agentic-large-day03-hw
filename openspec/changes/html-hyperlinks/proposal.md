# Proposal: HTML-like hyperlinks for text with `link`

## Чому

- Користувачі вводять markdown-посилання `[label](url)` у текст; після сабміту зберігаються **видимий текст** `label` і властивість **`link`**.
- На canvas текст досі виглядає як звичайний, а клік здебільшого очікується біля **іконки**, а не на підписі — це слабко відповідає очікуванню «гіперпосилання як у HTML» (issue #11024 / `docs/specs/11024.md`).

## Що

1. **Візуально**: для тексту з **вирішеним** URL (`element.link` після сабміту або одиночний валідний `[label](url)` у рядку) — колір `#1971c2` (узгоджено з іконкою hyperlink) і **підкреслення** по рядках на canvas; у SVG — `fill` + `text-decoration: underline` (зовнішній або внутрішній `<a>` залежно від `element.link`).
2. **Взаємодія**: для **тексту** з вирішеним URL, коли елемент **не вибраний**, клік по **bounding box** тексту вважається кліком по посиланні — ті самі правила, що й для `handleElementLinkClick` / `normalizeLink` / `onLinkOpen` (для тексту bbox застосовується і на touch form factor).
3. **Без зміни**: повна markdown-розмітка в редакторі; поведінка embeddable / інших типів елементів з `link`; UX **редагування** тексту (wysiwyg) — лише рендер і hit-test поза режимом редагування тексту.

## Вплив

| Область | Ризик / примітка |
|---------|------------------|
| `packages/common/src/markdownLink.ts` | Парсинг одного `[label](url)` на рядок + відхилення небезпечних схем. |
| `packages/element/src/renderElement.ts` | Зміна пікселів кешу тексту; потрібна узгодженість з темою (dark). |
| `packages/excalidraw/components/hyperlink/helpers.ts` | Розширення `isPointHittingLink`: текст з посиланням стає «клікабельним» по всьому bbox — вибір елемента одним кліком по тексту ускладнюється (альтернатива: marquee, клік поза текстом). |
| `staticSvgScene.ts` | Експортований SVG виглядає ближче до семантики посилання. |
| Тести | Можливі оновлення снапшотів / нові unit-тести на helpers або парсер (якщо додаються). |

## Ризики та мітигація

| Ризик | Мітигація |
|-------|-----------|
| Регресія рендеру тексту без посилання | `getTextHyperlinkRenderState` повертає `styleAsHyperlink: false`; snapshot / ручна перевірка. |
| Обхід санітизації URL | `parseMarkdownLink` відхиляє небезпечні схеми; далі `normalizeLink` як у решті редактора. |
| Вибір тексту кліком по підпису | Свідомий компроміс: коли не вибрано — bbox відкриває посилання; вибір через marquee або клік поза текстом (див. `decisionLog.md`). |
| Темна тема / контраст | Canvas використовує `applyDarkModeFilter(TEXT_HYPERLINK_COLOR)` поруч з іконками. |
| SVG експорт і подвійні `<a>` | У `staticSvgScene` внутрішній `<a>` лише коли немає зовнішнього від `element.link`. |
