# Delta: text-rendering — ADDED requirements

**Change:** `html-hyperlinks`  
**Проти baseline:** `openspec/specs/text-rendering/spec.md`

Нижче лише **нові або змінені** вимоги. Усе, що не згадано, залишається як у baseline.

## ADDED — Canvas text з `element.link`

### REQ-TR-LINK-001 — Колір посилання

Якщо `isTextElement(element)` і `element.link` визначений (truthy), колір заливки тексту на canvas **має** відрізнятися від звичайного `strokeColor`: використовується **погоджений колір посилання** (наприклад відтінок синього з палітри проєкту, узгоджений з UI hyperlink), з **коректною обробкою темної теми** (наприклад через той самий механізм, що й для інших елементів).

### REQ-TR-LINK-002 — Підкреслення

Для того самого випадку (`text` + `link`) кожен **непорожній** рядок відображеного `element.text` **має** мати **візуальне підкреслення** під базовою лінією рядка (наприклад лінія `stroke` після `fillText`), з узгодженим кольором і товщиною, масштабованою від `fontSize`. Порожні рядки не підкреслюються.

### REQ-TR-LINK-003 — Без `link`

Якщо `element.link` відсутній або falsy, рендер тексту **має** збігатися з baseline (колір `strokeColor`, **без** додаткового підкреслення з цієї фічі).

## ADDED — SVG text з `element.link`

### REQ-TR-LINK-004 — Семантика вигляду в експорті

Для текстових елементів з `element.link` SVG **має** відображати підпис як гіперпосилання за виглядом: **fill** як у REQ-TR-LINK-001 та **`text-decoration: underline`** на відповідних вузлах `<text>` (або еквівалент, погоджений з рушієм SVG), узгоджено з темою експорту.

## ADDED — Pointer / hit-test

### REQ-TR-LINK-005 — Клік по тексту посилання

Для `isTextElement(element)` з `element.link`, коли елемент **не** у вибраному стані (`selectedElementIds`), на **десктопі** (`!isMobile`) точка всередині **bounding box** тексту **має** вважатися попаданням у посилання для цілей відкриття URL (той самий шлях, що й для існуючого кліку по іконці / view mode), з урахуванням `normalizeLink` та `onLinkOpen`.

### REQ-TR-LINK-006 — Мобільні пристрої

На мобільних пристроях baseline-логіка (наприклад акцент на іконці або наявні touch-шляхи) **зберігається**, якщо не змінюється окремим рішенням продукту.

## Non-goals (без змін)

- Повний markdown у тексті.
- Зміна UX **редагування** тексту в wysiwyg.
- Зміна моделі embeddable або загальної семантики `link` для не-текстових елементів понад необхідне для сумісності.
